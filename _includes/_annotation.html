{% comment %} Shared internal annotation renderer for glossary and citation popovers. Purpose: - Provide a single markup contract for all inline annotations so CSS and JavaScript only target one structure. - Normalize accessibility hooks, trigger markup, popover shell markup, and shared content regions such as extra notes. Expected inputs from wrapper includes: - `kind`: required. Normalized variant name, typically `glossary` or `citation`. - `trigger_html`: required. Safe inline HTML used inside the trigger button. - `heading`: optional. Popover heading text. - `eyebrow`: optional. Small type label above the heading. - `body_html`: optional. Main popover body HTML. - `extra_html`: optional. One-off appended content shown in the shared "extra" slot. - `class`: optional. Extra class names added to the root wrapper. - `aria_label`: optional. Override for the trigger button's accessible label. - `style`: optional. Reserved for future rendering variants. - `source_id`: optional. Canonical source ID for citation hydration. - `citation_label`: optional. Page-local citation label for citation hydration/marker output. Behavior notes: - The root `[data-annotation]` element is the contract targeted by the site CSS/JS. - Without JavaScript, the popover still becomes visible on hover and focus via CSS. - With JavaScript enabled, click/tap toggling, outside-click dismissal, Escape dismissal, and viewport-aware positioning are added.
