{% comment %} Author-facing citation annotation include. Purpose: - Pull canonical citation records from `_data/citation.json`. - Resolve page-local labels such as `label="3"` through the current page's front matter `citations` map or array when available. - Delegate all shared annotation markup and interaction hooks to `_annotation.html`. Parameters: - `label`: optional page-local citation label. Preferred for Jekyll 3 compatibility. - `n`: optional page-local citation label alias. Supported on a best-effort basis. - `id`: optional direct canonical citation ID fallback from `_data/citation.json`. - `text`: optional visible trigger text override. If omitted, numbered superscript markers are used when a label exists. - `extra`: optional one-off appended content. - `style`: optional citation rendering style. Defaults to `site`. - `class`: optional extra root class names. Supported front matter structures: - Hash mapping, convenient for short numeric labels: --- citations: "1": hall-2022-metabolism "2": schoenfeld-2018-protein --- - Ordered array, useful when you want explicit sequence control or non-numeric labels: --- citations: - label: "1" id: hall-2022-metabolism - label: "2" id: schoenfeld-2018-protein --- Example usage: - `{% include citation.html label="3" %}` - `{% include citation.html label="3" extra="Cited here specifically for the adaptive thermogenesis discussion." %}` - `{% include citation.html id="hall-2022-metabolism" label="7" %}` Notes: - If both `id` and `label`/`n` are provided, the explicit `id` wins. - On Jekyll 3, `label` is the more reliable inline parameter and should be preferred in authored content. - Citation links default to opening in a new tab. Data fields such as `url_newtab`, `doi_newtab`, and `links[].newtab` can override that behavior. - The current default renderer is `site`; `chicago` is accepted as an alias so future style-specific
